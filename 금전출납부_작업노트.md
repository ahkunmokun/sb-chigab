# 금전출납부 · Kassenbuch — 작업 노트

작성일: 2026-06-26

## 만든 것
베를린 거주 한국인 프리랜서/예술감독을 위한 **이중 통화 금전출납부**.
단일 HTML 파일, 서버 불필요, 브라우저만으로 실행, 폰 URL 접속 가능.

## 산출 파일

| `ledger.html` | 앱 본체 (단일 파일, 인라인 CSS/JS) |
| `README.md` | 사용법 + GitHub Pages 배포법 |
| `금전출납부_작업노트.md` | 이 노트 |

## 기술 스택
- **OCR**: Tesseract.js v5 (CDN), kor+deu+eng 동시 로드, 비동기
- **환율**: Frankfurter API (`base=EUR&quotes=KRW,USD`), ECB 기준, 무료·키없음
- **저장**: 브라우저 localStorage (`raee_ledger_v1`, `raee_fx_cache`)
- 다크모드 자동, 모바일 최우선 반응형

## 구현 기능
### 영수증 스캔
- 폰 카메라(`capture="environment"`) / PC 드래그앤드롭·파일선택
- 날짜(YYYY-MM-DD·DD.MM.YYYY·YYYY.MM.DD)·금액·통화·카테고리 자동 파싱
- 키워드 매칭: KSK, Miete/집세, Strom/전기, Handy/통신, Millie/밀리, Claude/Anthropic, Krankenkasse/건강보험
- 신뢰도 60% 미만 경고
- 이미지 OCR 직후 `revokeObjectURL`로 즉시 해제 (저장 안 함)

### 환율
- 하루 1회 캐시, 오프라인 시 마지막 값 사용
- 크로스 환율: `usdKrw = rates.KRW / rates.USD`
- 수동 갱신 / 자동 중단 토글 / 값 직접 입력
- 기본값: 1 EUR = 1,480 KRW / 1 USD = 1,370 KRW

### 탭 3개
1. **거래 기록** — 스캔 버튼(하단 강조) + 수동 입력 + 요약카드(EUR/KRW 수입·지출, EUR 환산 순이익, 건수) + 월/유형/통화/국가 필터 테이블
2. **세금 내보내기** — 독일 EÜR / Kassenbuch(GoBD 잔액누적) / 한국 종합소득세, 전부 UTF-8 BOM CSV
3. **설정** — 환율, 집세 사업용 비율(독/한 각 0%, 2026.04 변경 예정), 카테고리 추가/삭제, JSON 백업/복원/전체삭제

## 추가된 기능 (2차)
- 저축: 유형 '저축' + 저축 통(세금대비/비상금 등)별 입금·출금 + 목표 진행률 바
- 기부: 유형 '기부', 독일 Spenden/한국 기부금 별도 집계 + 전용 CSV (사업손익과 분리)
- 식료품(장보기) 카테고리 DE/KR 추가 + OCR 키워드(REWE·이마트 등)
- 사진 첨부: 거래에 영수증 원본 압축 저장(최대 1000px) + 목록 📷로 열람
- OCR 전처리: 축소+흑백+대비강화, PSM6 — 인식률 개선 (그래도 완벽치 않음, 수동확인 필수)

## 추가된 기능 (3차)
- 환율 버그 수정: Frankfurter v2가 배열 응답이라 `d.rates.KRW` 파싱 실패 → 1480 고정이었음. 배열/객체 모두 파싱하도록 수정. 기본값 1755/1543로 현실화, 갱신시각 표시. (실제 ECB: 1€≈1755원)
- 기기 간 동기화(Supabase): 전체 state를 jsonb 한 행으로, rev 시각 기준 LWW. 이메일+비번 로그인. schedule-app과 같은 프로젝트(hbirnbnoqaoqizlrphoi) 사용, 테이블 `ledger_state`. publishable 키 공개앱 내장(RLS로 보호).
- 사용자 수동 설정 필요: 동기화_설정안내.md 참고 (SQL 1회 + 이메일확인 OFF)

## 추가된 기능 (4차)
- 영수증 스캔에 **사진첩 선택** 추가: 기존엔 스캔 버튼이 `capture="environment"`로 카메라만 열렸음. `capture` 없는 입력(`fileInputGallery`) + "🖼 사진첩에서 영수증 불러와 스캔" 버튼 추가, 동일한 `runOCR(f)` 재사용. (촬영 못한 이미 찍어둔 영수증도 스캔 가능)
